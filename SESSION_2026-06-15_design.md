## ⭐ 워크플로우 대전환 — Claude Code 자율 개발 시스템

이번 세션의 가장 큰 결정. 주진님이 "매번 시안 보고, 빌드 기다리고, 스크린샷 찍어 확인하는 과정"이 병목이라고 판단.

### 목표 재정의
- ❌ "OverClan을 자동으로 만든다" (1회성)
- ✅ **"어떤 프로젝트에도 붙일 재사용 가능한 자율 개발 시스템"** 구축 → OverClan은 1호 검증 대상
- 향후 적용 예정: 자동매매 프로그램, 그로브(매매일지 앱), 로블록스 게임 개발

### 주진님의 새 역할 (①번 선택)
> 집 PC를 작업 서버로 켜둠 → 에이전트가 알아서 개발/확인/배포 → 폰으로 **보고만 받고 "A/B 골라줘"에 답만**

### 시스템 구조 (목표)
```
[집 PC = 항상 켜진 작업 서버]
├── Orchestrator (메인 Claude Code) — 작업 분배/감시/병합, 애매하면 텔레그램으로 "A/B?"
├── Developer 에이전트 — 코드 수정→빌드→배포
├── QA 에이전트 (Chrome/Preview 연동) — 사이트 직접 띄워 스크린샷, 버그 자가 발견
└── 작업 큐 (파일 기반)
        ↓ 텔레그램 알림
[폰/회사 PC] ← 주진님: 보고 받고 선택만
```

### 구축 단계 (점진적)
1. **[완료] Chrome/Preview 연동** — Claude Code가 OverClan을 직접 띄워 스크린샷·SKIP클릭까지 자가 확인 성공 (2026-06-15 검증됨)
2. **[다음] 텔레그램 봇** — 폰으로 "A/B?" 보고 받기
3. 자동 루프 — 수정→빌드→자가확인→배포 자동화
4. 역할 분리(QA/Developer) → 병렬화
5. 템플릿화 → 다음 프로젝트에 복붙
